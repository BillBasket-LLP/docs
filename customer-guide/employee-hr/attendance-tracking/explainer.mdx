Mark attendance for each employee, every day, from a simple date-picker
screen. Attendance feeds directly into payroll, so a few seconds of daily
entry keeps salary calculations accurate at month-end.

<Note>
  Employee HR is an optional add-on module — turn it on from Settings →
  Add-ons (Admin PIN required) if you don't see it in your menu.
</Note>

## Understanding the fields

- **Date** — the day you're marking attendance for. One row is stored per
  employee per day, so you can go back and fix an earlier date if you forget
  to mark it same-day.
- **Status** — each employee's day is set to one of **Present**, **Absent**,
  **Half-day**, **Paid Leave**, **Week off**, or **Holiday**. The last two
  aren't things you pick by hand day-to-day — a date you've added under
  [Holidays](/customer-guide/employee-hr/holidays/explainer) is automatically
  reflected here, and your store's weekly off day (set separately in
  Settings) works the same way.
- **Mark all present** — a shortcut that sets every active employee's status
  for the selected date to Present in one tap, since most days most staff
  simply show up. Selecting an individual employee afterward and choosing a
  different status overrides the "all present" default only for that one
  person.
- **Hours worked** — an optional number of hours for the day, if you want
  finer-grained tracking than a simple Present/Half-day split.
- **Note** — a free-text field for context, like "left early — doctor's
  appointment."
- **Joining date** — set on the employee's own record, not here — attendance
  and payroll both ignore any date before an employee's joining date, so a
  newly added employee doesn't accidentally show as "absent" for days before
  they started.

<Note>
  Only an employee's **status** for each day feeds payroll — see
  [Payroll & salary](/customer-guide/employee-hr/payroll-and-salary/explainer)
  for exactly how Present/Absent/Half-day/Paid Leave/Holiday days turn into
  payable days.
</Note>
